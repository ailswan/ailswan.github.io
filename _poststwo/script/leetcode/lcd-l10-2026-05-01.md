Topological sort is used when we need to order tasks with dependencies.

The basic idea is:

```text
If A must happen before B,

```

Common examples include course prerequisites, build systems, package dependencies, task scheduling, workflow execution, and alien dictionary ordering.

**Topological Sort is about dependency ordering.**

## PPT 3 — Directed Acyclic Graph

Topological sort applies to a **Directed Acyclic Graph**, usually called a DAG.

Directed means edges have direction.

Acyclic means there are no cycles.

For example:

```text
A → B
A → C
B → D
C → D
```

Valid topological orders include:

```text
A, B, C, D
A, C, B, D
```

Both are valid because A comes before B and C, and both B and C come before D.

But this order is invalid:

```text
B, A, C, D
```

because A must come before B.

The key idea is:

**An edge from A to B means A must appear before B.**

**(PPT: DAG Example — Valid vs Invalid Order)**

## PPT 4 — Why Topological Sort Works

Topological sort works by repeatedly processing nodes that have no remaining prerequisites.

We represent this using **indegree**.

```text
indegree[node] = number of prerequisites remaining
```

If a node has indegree zero, it can be processed immediately.

After processing it, we remove its outgoing edges and reduce the indegree of its neighbors.

For example:

```text
A → B
A → C
B → D
C → D
```

Initial indegree is:

```text
A: 0
B: 1
C: 1
D: 2
```

We process A first.

Then B and C become available.

After processing B and C, D becomes available.

The key idea is:

**A node is ready when all of its prerequisites are already processed.**

**(PPT: Indegree and Dependency Readiness)**

## PPT 5 — Kahn’s Algorithm

The most common implementation is **Kahn’s Algorithm**.

It uses an indegree array and a queue.

```python
from collections import deque

def topological_sort(n, edges):
    graph = [[] for _ in range(n)]
    indegree = [0] * n

    for pre, nxt in edges:
        graph[pre].append(nxt)
        indegree[nxt] += 1

    queue = deque()

    for node in range(n):
        if indegree[node] == 0:
            queue.append(node)

    order = []

    while queue:
        node = queue.popleft()
        order.append(node)

        for neighbor in graph[node]:
            indegree[neighbor] -= 1

            if indegree[neighbor] == 0:
                queue.append(neighbor)

    if len(order) != n:
        return []

    return order
```

If the final order does not contain all nodes, that means some nodes were blocked forever.

So the graph has a cycle.

The key idea is:

**Kahn’s Algorithm processes nodes in dependency-ready order.**

**(PPT: Kahn’s Algorithm — Indegree + Queue)**

## PPT 6 — Edge Direction Matters

One of the most common bugs is building edges in the wrong direction.

For Course Schedule, the input usually means:

```text
[course, prerequisite]
```

So if `b` is a prerequisite of `a`, the edge should be:

```text
b → a
```

Because after completing `b`, we unlock `a`.

In code:

```python
graph[prereq].append(course)
indegree[course] += 1
```

The wrong direction would be:

```python
graph[course].append(prereq)
```

That reverses the dependency order.

The key idea is:

**Always define what an edge means before building the graph.**

**(PPT: Edge Direction — prerequisite → course)**

## PPT 7 — Course Schedule

Course Schedule is the classic topological sort problem.

The question is:

Can we finish all courses?

This is really asking:

Does the prerequisite graph have a cycle?

```python
from collections import deque

def can_finish(num_courses, prerequisites):
    graph = [[] for _ in range(num_courses)]
    indegree = [0] * num_courses

    for course, prereq in prerequisites:
        graph[prereq].append(course)
        indegree[course] += 1

    queue = deque([i for i in range(num_courses) if indegree[i] == 0])
    completed = 0

    while queue:
        course = queue.popleft()
        completed += 1

        for next_course in graph[course]:
            indegree[next_course] -= 1

            if indegree[next_course] == 0:
                queue.append(next_course)

    return completed == num_courses
```

If all courses are processed, there is no cycle.

If not, some courses depend on each other, so completion is impossible.

The key idea is:

**Course Schedule is directed cycle detection through topological sort.**

## PPT 8 — DFS-Based Topological Sort

Topological sort can also be done with DFS.

The idea is:

```text
Visit all descendants first,
then add the current node to the result.
```

This is postorder.

Finally, we reverse the result.

DFS also uses three states for cycle detection:

```text
0 = unvisited
1 = visiting
2 = visited
```

If DFS reaches a node that is currently visiting, we found a cycle.

```python
def topo_sort_dfs(n, edges):
    graph = [[] for _ in range(n)]

    for pre, nxt in edges:
        graph[pre].append(nxt)

    state = [0] * n
    order = []

    def dfs(node):
        if state[node] == 1:
            return False
        if state[node] == 2:
            return True

        state[node] = 1

        for neighbor in graph[node]:
            if not dfs(neighbor):
                return False

        state[node] = 2
        order.append(node)
        return True

    for node in range(n):
        if state[node] == 0:
            if not dfs(node):
                return []

    order.reverse()
    return order
```

The key idea is:

**DFS topological sort is reverse postorder with cycle detection.**

**(PPT: DFS Topological Sort)**

## PPT 9 — Alien Dictionary and Multiple Valid Orders

Alien Dictionary is another common topological sort problem.

Each character is a node.

We compare adjacent words and find the first different character.

For example:

```text
wrt
wrf
```

The first difference is:

```text
t before f
```

So we create an edge:

```text
t → f
```

Then we run topological sort on characters.

There is also an important invalid case:

```text
["abc", "ab"]
```

A longer word cannot appear before its own prefix.

Another important point is that topological order may not be unique.

If multiple nodes have indegree zero, any of them can come next.

If the problem asks for the lexicographically smallest order, we use a min-heap instead of a queue.

The key idea is:

**Topological sort may return one valid order, not necessarily the only order.**

**(PPT: Alien Dictionary + Multiple Valid Orders)**

## PPT 10 — Senior-Level Explanation and Closing Insight

At senior level, I would explain topological sort like this.

Topological sort is used to order nodes in a directed dependency graph.

If there is an edge from A to B, then A must appear before B.

A valid topological order only exists if the graph is a DAG.

The most common implementation is Kahn’s Algorithm.

It uses indegree to track how many prerequisites each node still has.

Nodes with indegree zero are ready to process.

After processing a node, we reduce the indegree of its neighbors.

If all nodes are processed, we have a valid order.

If not, the graph contains a cycle.

DFS-based topological sort is another approach.

It uses postorder traversal and three states to detect cycles.

The complexity is usually:

```text
Time: O(V + E)
Space: O(V + E)
```

The final insight is:

**Topological Sort is not just sorting.
It is dependency readiness.**

If you can explain what the nodes represent, what the directed edges mean, how indegree tracks remaining prerequisites, and why cycles make ordering impossible, then you understand topological sort at an interview-ready level.
