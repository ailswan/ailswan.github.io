## PPT 3 — DFS Traversal Overview

DFS means depth-first search.

In a binary tree, DFS usually appears in three classic forms:

```text
Preorder:  root, left, right
Inorder:   left, root, right
Postorder: left, right, root
```

The recursive shape is almost always similar:

```python
def dfs(node):
    if not node:
        return

    dfs(node.left)
    dfs(node.right)
```

The difference is where we process the current node.

If we process the node before children, it is preorder.

If we process it between left and right, it is inorder.

If we process it after children, it is postorder.

The key idea is:

**DFS traversal order is decided by where we place the process step.**

**(PPT: DFS — Preorder / Inorder / Postorder)**

## PPT 4 — Preorder Traversal

Preorder traversal means:

```text
Root
Left
Right
```

It processes the parent before children.

This is useful when parent information needs to be handled first.

Common use cases include serialization, copying a tree, constructing a tree, and top-down path tracking.

```python
def preorder_traversal(root):
    result = []

    def dfs(node):
        if not node:
            return

        result.append(node.val)
        dfs(node.left)
        dfs(node.right)

    dfs(root)
    return result
```

The key idea is:

**Preorder is useful when the current node should be processed before its subtrees.**

**(PPT: Preorder — Root Before Children)**

## PPT 5 — Inorder Traversal

Inorder traversal means:

```text
Left
Root
Right
```

For a general binary tree, this is just one traversal order.

But for a binary search tree, inorder traversal has a special meaning.

It visits values in sorted order.

That makes it useful for BST validation, kth smallest element, and problems that depend on sorted BST order.

```python
def inorder_traversal(root):
    result = []

    def dfs(node):
        if not node:
            return

        dfs(node.left)
        result.append(node.val)
        dfs(node.right)

    dfs(root)
    return result
```

The key idea is:

**Inorder traversal gives sorted order only for a valid BST, not for every binary tree.**

**(PPT: Inorder — BST Sorted Order)**

## PPT 6 — Postorder Traversal

Postorder traversal means:

```text
Left
Right
Root
```

It processes children before the current node.

This is very useful for bottom-up tree problems.

Examples include maximum depth, balanced binary tree, diameter of binary tree, subtree size, and subtree validation.

```python
def postorder_traversal(root):
    result = []

    def dfs(node):
        if not node:
            return

        dfs(node.left)
        dfs(node.right)
        result.append(node.val)

    dfs(root)
    return result
```

The key idea is:

**Postorder is useful when the parent depends on results from its children.**

**(PPT: Postorder — Children Before Parent)**

## PPT 7 — Level Order Traversal

Level order traversal visits nodes level by level.

This is BFS, not DFS.

We use a queue.

At the start of each loop, the queue contains all nodes in the current level.

```python
from collections import deque

def level_order(root):
    if not root:
        return []

    result = []
    queue = deque([root])

    while queue:
        level_size = len(queue)
        level = []

        for _ in range(level_size):
            node = queue.popleft()
            level.append(node.val)

            if node.left:
                queue.append(node.left)

            if node.right:
                queue.append(node.right)

        result.append(level)

    return result
```

Level order is useful for level output, zigzag traversal, right side view, width, and minimum depth.

The key idea is:

**BFS is the natural choice when the problem cares about levels.**

**(PPT: Level Order — BFS with Queue)**

## PPT 8 — Recursive vs Iterative Traversal

Recursive traversal is usually the cleanest way to solve tree problems.

But iterative traversal is also important when the tree is very deep and recursion depth may become a problem.

For iterative preorder, we use a stack.

Because stack is last-in-first-out, we push the right child before the left child.

```python
def preorder_iterative(root):
    if not root:
        return []

    result = []
    stack = [root]

    while stack:
        node = stack.pop()
        result.append(node.val)

        if node.right:
            stack.append(node.right)

        if node.left:
            stack.append(node.left)

    return result
```

For iterative inorder, we keep pushing left nodes until null, then process the node and move right.

The key idea is:

**Recursion uses the call stack; iteration uses an explicit stack or queue.**

**(PPT: Recursive vs Iterative Traversal)**

## PPT 9 — Path Problems and Bottom-Up Recursion

Tree path problems often use top-down DFS.

For example, Path Sum passes the remaining target down the tree.

```python
def has_path_sum(root, target_sum):
    if not root:
        return False

    if not root.left and not root.right:
        return target_sum == root.val

    remaining = target_sum - root.val

    return (
        has_path_sum(root.left, remaining)
        or
        has_path_sum(root.right, remaining)
    )
```

If we need to return all paths, we use backtracking.

We append the current node, explore children, and pop before returning.

Bottom-up problems are different.

For maximum depth:

```text
current depth = 1 + max(left depth, right depth)
```

For diameter:

```text
candidate diameter = left height + right height
```

The key idea is:

**Top-down DFS passes state downward; bottom-up recursion returns information upward.**

**(PPT: Path Problems + Bottom-Up Recursion)**

## PPT 10 — Senior-Level Explanation and Closing Insight

At senior level, I would explain tree traversal like this.

Tree traversal is the process of visiting nodes in a controlled order.

Preorder processes the current node before children.

It is useful for serialization, copying, construction, and top-down path logic.

Inorder processes left subtree, current node, then right subtree.

For BSTs, it gives sorted order.

Postorder processes children before the current node.

It is useful for bottom-up problems where the parent depends on child results, such as depth, balanced tree, diameter, and subtree computations.

Level order uses BFS and processes nodes level by level.

It is useful when the problem asks for levels, width, right side view, zigzag traversal, or minimum depth.

The most important design question is:

```text
What should each recursive call return?
```

The final insight is:

**Tree Traversal Patterns are not just memorizing preorder, inorder, and postorder.
They are about deciding when the current node should be processed.**

If you can explain what the node needs from its parent, what it needs from its children, what the recursive function returns, and whether the problem needs DFS or BFS, then you understand tree traversal at an interview-ready level.

Thank you.
