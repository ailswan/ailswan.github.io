## PPT 1 — Title Slide

Hello everyone.
Today I’d like to walk through one of the most important algorithm patterns: **Binary Search**.

Binary search is not just about searching in a sorted array.

At a deeper level, binary search is about finding a boundary in a search space that has a monotonic property.

The basic flow looks like this:

```text
Define search space
→ Pick the middle point
→ Check the condition
→ Safely discard one half
→ Repeat until the boundary is found
```

The key idea is:

**Each check eliminates half of the search space.**

That is why binary search can reduce a linear **O(n)** search into **O(log n)**.

## PPT 3 — Why Binary Search Works

A brute-force search scans every element:

```python
for i in range(len(nums)):
    if nums[i] == target:
        return i
```

This takes **O(n)** time.

If the array is sorted, we can compare the middle element with the target.

```text
nums[mid] < target → search the right half
nums[mid] > target → search the left half
```

Why is this safe?

Because in a sorted array, if `nums[mid]` is smaller than the target, then everything to the left of `mid` is also too small.

So we can discard that entire half.

The key requirement is:

```text
Can I safely discard one side?
```

This usually comes from sorted order, a monotonic predicate, or an ordered answer space.

**(PPT: Linear Search O(n) → Binary Search O(log n))**

## PPT 4 — Classic Binary Search

The classic version searches for an exact target in a sorted array.

```python
def binary_search(nums, target):
    left = 0
    right = len(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2

        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return -1
```

Here, I use an inclusive range.

That means both `left` and `right` are valid candidate positions.

So the loop condition is:

```text
left <= right
```

If the middle value is too small, I move `left` to `mid + 1`.

If the middle value is too large, I move `right` to `mid - 1`.

The key idea is:

**For exact search, exclude mid after checking it.**

**(PPT: Classic Binary Search — Inclusive Range)**

## PPT 5 — Lower Bound and Upper Bound

Boundary search is where binary search becomes very powerful.

Lower bound means:

```text
Find the first index where nums[index] >= target
```

The code is:

```python
def lower_bound(nums, target):
    left = 0
    right = len(nums)

    while left < right:
        mid = left + (right - left) // 2

        if nums[mid] >= target:
            right = mid
        else:
            left = mid + 1

    return left
```

Upper bound means:

```text
Find the first index where nums[index] > target
```

The difference is the condition.

Lower bound uses `>= target`.

Upper bound uses `> target`.

With these two helpers, we can find the first and last occurrence of a target.

```text
first occurrence = lower_bound(target)
last occurrence = upper_bound(target) - 1
```

The key idea is:

**Boundary search keeps mid when mid could still be the answer.**

**(PPT: Lower Bound vs Upper Bound)**

## PPT 6 — Search Insert Position

Search insert position is basically lower bound.

The problem asks:

Where should the target be inserted in a sorted array?

For example:

```text
nums = [1, 3, 5, 6]

target = 5 → answer = 2
target = 2 → answer = 1
target = 7 → answer = 4
```

The code is:

```python
def search_insert(nums, target):
    left = 0
    right = len(nums)

    while left < right:
        mid = left + (right - left) // 2

        if nums[mid] >= target:
            right = mid
        else:
            left = mid + 1

    return left
```

If target exists, this returns its index.

If target does not exist, this returns the first position where it can be inserted while keeping the array sorted.

The key idea is:

**Search insert position is lower bound in disguise.**

**(PPT: Search Insert = Lower Bound)**

## PPT 7 — Binary Search on Answer

Binary search is not limited to arrays.

Sometimes the array itself is not sorted, but the answer space is monotonic.

This is called **binary search on answer**.

The pattern is:

```text
Define the answer range
Define feasible(x)
Find the smallest x that satisfies feasible(x)
```

For example, in Koko Eating Bananas, the answer is eating speed.

If speed `k` works, then any larger speed also works.

So the validity pattern is:

```text
false false false true true true
```

We want the first true.

The template is:

```python
def binary_search_answer(left, right):
    while left < right:
        mid = left + (right - left) // 2

        if feasible(mid):
            right = mid
        else:
            left = mid + 1

    return left
```

The key idea is:

**If feasibility changes monotonically, we can binary search the answer.**

**(PPT: Binary Search on Answer — First True)**

## PPT 8 — Example: Koko Eating Bananas

In Koko Eating Bananas, we want the minimum eating speed so Koko can finish all piles within `h` hours.

The search range is:

```text
left = 1
right = max(piles)
```

The feasibility check asks:

Can Koko finish with this speed?

```python
def min_eating_speed(piles, h):
    left = 1
    right = max(piles)

    def feasible(speed):
        hours = 0

        for pile in piles:
            hours += (pile + speed - 1) // speed

        return hours <= h

    while left < right:
        mid = left + (right - left) // 2

        if feasible(mid):
            right = mid
        else:
            left = mid + 1

    return left
```

If a speed works, a larger speed also works.

So we search for the smallest feasible speed.

The key idea is:

**The hard part is not the binary search code. The hard part is defining the monotonic feasible function.**

**(PPT: Koko Eating Bananas — Feasible Function)**

## PPT 9 — Rotated Array and Peak Finding

Binary search can also work when the array is not globally sorted, as long as there is still useful structure.

For a rotated sorted array:

```text
nums = [4, 5, 6, 7, 0, 1, 2]
```

At each step, at least one half is sorted.

So we first decide whether the left half or right half is sorted, then check whether the target lies inside that sorted half.

For finding the minimum in a rotated sorted array, we compare:

```text
nums[mid] with nums[right]
```

If `nums[mid] > nums[right]`, the minimum is on the right.

Otherwise, the minimum is at `mid` or on the left.

For peak element, we compare:

```text
nums[mid] with nums[mid + 1]
```

If we are going uphill, a peak exists on the right.

If we are going downhill, a peak exists at mid or on the left.

The key idea is:

**Binary search only needs a safe direction decision, not always a fully sorted array.**

**(PPT: Rotated Array + Peak Finding)**

## PPT 10 — Senior-Level Explanation and Closing Insight

At senior level, I would explain binary search like this.

Binary search is a technique for searching a sorted or monotonic search space.

At each step, we check the middle point and use the result to safely discard half of the search space.

For exact target search, I usually use an inclusive range with `left <= right`.

For boundary search, such as lower bound, upper bound, search insert position, or first bad version, I use a first-true or half-open template.

For binary search on answer, I define a feasible function.

If the predicate changes from false to true, I search for the first true.

If it changes from true to false, I search for the last true.

The main risks are off-by-one errors, unclear boundary definitions, wrong loop conditions, infinite loops, and using binary search when the predicate is not monotonic.

The final insight is:

**Binary Search is not about finding the middle.
It is about finding a monotonic boundary.**

If you can explain the search space, the predicate, the invariant, and why one half can be discarded safely, then you understand binary search at an interview-ready level.

Thank you.

**(PPT: Closing Insight — Monotonic Boundary)**
