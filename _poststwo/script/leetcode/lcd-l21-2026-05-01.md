## PPT 2 — Core Framework

When I think about interval problems, I usually break them down into five parts.

First, what the interval represents.
Second, whether the boundary is inclusive or half-open.
Third, how to detect overlap.
Fourth, whether the problem needs sorting, greedy, heap, or sweep line.
And finally, how to handle edge cases.

The most important question is:

```text
Do touching intervals overlap?
```

For meeting rooms, intervals are usually half-open.

```text
[start, end)
```

So a meeting ending at 10 does not conflict with another meeting starting at 10.

For merge intervals, boundaries are often treated as inclusive.

The key idea is:

**Before writing code, define the interval semantics.**

## PPT 3 — Overlap Logic

For two intervals:

```text
a = [a_start, a_end]
b = [b_start, b_end]
```

For inclusive intervals, they overlap if:

```text
a_start <= b_end and b_start <= a_end
```

Another way to think about it is:

Two intervals do not overlap if one ends before the other starts.

```text
a_end < b_start
or
b_end < a_start
```

For half-open meeting intervals, non-overlap becomes:

```text
a_end <= b_start
or
b_end <= a_start
```

That small equal sign difference matters a lot.

The key idea is:

**Most interval bugs come from the wrong overlap condition.**

**(PPT: Inclusive vs Half-Open Overlap)**

## PPT 4 — Why Sorting Helps

Without sorting, intervals can appear in any order.

```text
[8, 10], [1, 3], [2, 6]
```

It is hard to know which intervals should be compared.

After sorting by start time:

```text
[1, 3], [2, 6], [8, 10]
```

overlapping intervals usually become adjacent.

That allows us to scan from left to right and only compare with the previous interval, the last merged interval, or active intervals in a heap.

Most interval problems begin with:

```python
intervals.sort(key=lambda x: x[0])
```

The key idea is:

**Sorting turns scattered ranges into an ordered timeline.**

**(PPT: Unsorted Intervals → Sorted Timeline)**

## PPT 5 — Merge Intervals

A classic interval problem is **Merge Intervals**.

Given:

```text
[[1, 3], [2, 6], [8, 10], [15, 18]]
```

The output is:

```text
[[1, 6], [8, 10], [15, 18]]
```

The algorithm is simple.

Sort by start time.

Then scan intervals one by one.

```python
def merge(intervals):
    intervals.sort(key=lambda x: x[0])

    result = []

    for start, end in intervals:
        if not result or result[-1][1] < start:
            result.append([start, end])
        else:
            result[-1][1] = max(result[-1][1], end)

    return result
```

If the current interval does not overlap with the last merged interval, append it.

Otherwise, merge by updating the end.

The key idea is:

**After sorting by start, merging only needs the last merged interval.**

**(PPT: Merge Intervals)**

## PPT 6 — Insert Interval

Insert Interval is another common pattern.

We are given sorted, non-overlapping intervals, and we need to insert a new interval.

The solution has three phases.

First, add all intervals that end before the new interval starts.

Second, merge all intervals that overlap with the new interval.

Third, add the remaining intervals.

```python
def insert(intervals, new_interval):
    result = []
    i = 0
    n = len(intervals)

    new_start, new_end = new_interval

    while i < n and intervals[i][1] < new_start:
        result.append(intervals[i])
        i += 1

    while i < n and intervals[i][0] <= new_end:
        new_start = min(new_start, intervals[i][0])
        new_end = max(new_end, intervals[i][1])
        i += 1

    result.append([new_start, new_end])

    while i < n:
        result.append(intervals[i])
        i += 1

    return result
```

The key idea is:

**Insert Interval is before, overlap, and after.**

**(PPT: Insert Interval — Three Phases)**

## PPT 7 — Meeting Rooms and Min Heap

For **Meeting Rooms I**, we check whether a person can attend all meetings.

Sort by start time.

If the current meeting starts before the previous meeting ends, there is a conflict.

```python
def can_attend_meetings(intervals):
    intervals.sort(key=lambda x: x[0])

    for i in range(1, len(intervals)):
        if intervals[i][0] < intervals[i - 1][1]:
            return False

    return True
```

For **Meeting Rooms II**, we need the minimum number of rooms.

That means we need the maximum number of active meetings at the same time.

We use a min heap of end times.

```python
import heapq

def min_meeting_rooms(intervals):
    intervals.sort(key=lambda x: x[0])
    heap = []

    for start, end in intervals:
        if heap and heap[0] <= start:
            heapq.heappop(heap)

        heapq.heappush(heap, end)

    return len(heap)
```

The heap tracks the earliest ending active meeting.

The key idea is:

**A min heap helps track active intervals by earliest end time.**

**(PPT: Meeting Rooms I + II)**

## PPT 8 — Sweep Line and Difference Array

Sweep line is another major interval pattern.

It converts intervals into events.

For each interval:

```text
start event: +1
end event: -1
```

Then we sort events and scan from left to right.

```python
def sweep_line(intervals):
    events = []

    for start, end in intervals:
        events.append((start, 1))
        events.append((end, -1))

    events.sort(key=lambda x: (x[0], x[1]))

    active = 0
    max_active = 0

    for time, delta in events:
        active += delta
        max_active = max(max_active, active)

    return max_active
```

Tie-breaking matters.

For half-open intervals, end events should be processed before start events at the same time.

If coordinates are small and bounded, a difference array can be even simpler.

For interval `[start, end)`:

```text
diff[start] += value
diff[end] -= value
```

The key idea is:

**Sweep line counts active intervals over time.**

**(PPT: Sweep Line + Difference Array)**

## PPT 9 — Greedy Interval Problems

Some interval problems are greedy.

For example:

**Non-overlapping Intervals** asks for the minimum number of intervals to remove.

A common greedy rule is:

```text
Sort by end time.
Keep the interval that ends earliest.
```

Why?

Because the earliest ending interval leaves the most room for future intervals.

The same idea appears in **Minimum Number of Arrows to Burst Balloons**.

Sort balloons by end position.

Shoot an arrow at the earliest end.

If the next balloon starts after the arrow position, we need a new arrow.

```python
def find_min_arrow_shots(points):
    if not points:
        return 0

    points.sort(key=lambda x: x[1])

    arrows = 1
    arrow_position = points[0][1]

    for start, end in points[1:]:
        if start > arrow_position:
            arrows += 1
            arrow_position = end

    return arrows
```

The key idea is:

**For interval selection, sorting by end time preserves future flexibility.**

**(PPT: Greedy Intervals — End Time Sorting)**

## PPT 10 — Senior-Level Explanation and Closing Insight

At senior level, I would explain interval problems like this.

Interval problems deal with ranges represented by start and end values.

The first step is to clarify interval semantics.

For meetings, intervals are usually half-open, so touching boundaries do not overlap.

For merge-style problems, boundaries are often inclusive.

Then I identify the problem type.

If the task is merging ranges, I sort by start time and scan.

If the task is inserting a new interval, I split the process into before, overlap, and after.

If the task is counting active intervals, I use a min heap or sweep line.

If the task is choosing intervals to keep or remove, I often sort by end time and apply greedy reasoning.

If coordinates are small and bounded, difference array can be a discrete sweep line.

The final insight is:

**Interval Problems are not just “sort and merge.”
They are about choosing the right model for overlap, coverage, active count, or scheduling.**

If you can explain interval semantics, the overlap condition, why sorting helps, when to use heap, sweep line, greedy, or difference array, and how boundary cases are handled, then you understand interval problems at an interview-ready level.
