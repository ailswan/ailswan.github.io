## PPT 3 — What Problem Are We Solving?

Two pointers is useful when we need to scan arrays, strings, or linked lists efficiently.

It is commonly used for pair search, palindrome checking, reversing data, removing duplicates, moving zeros, detecting cycles, and finding subarrays.

Many of these problems start with a brute-force nested loop.

For example, checking every pair usually looks like:

```text
for each i:
    for each j:
        check pair
```

That gives us **O(n²)** time.

Two pointers tries to replace this with a single scan.

Instead of checking every possible pair, we move the pointers based on the structure of the problem.

The key idea is:

**Two pointers works when pointer movement can safely eliminate part of the search space.**

**(PPT: Brute Force O(n²) → Two Pointers O(n))**

## PPT 4 — Why Two Pointers Work

Two pointers usually works when the data has some useful property.

The most common property is sorted order.

For example, in a sorted array, if we have one pointer at the left and one pointer at the right, then moving the left pointer usually increases the value, and moving the right pointer usually decreases the value.

So for a sorted two-sum problem:

```text
If sum is too small, move left.
If sum is too large, move right.
```

This is powerful because each pointer only moves in one direction.

That means every element is visited at most a constant number of times.

So the complexity becomes **O(n)** instead of **O(n²)**.

The key idea is:

**Two pointers is not magic. It works because the movement rule safely reduces the search space.**

**(PPT: Sorted Array Movement Rule)**

## PPT 5 — Main Two Pointer Patterns

There are four major two-pointer patterns.

The first is **opposite-direction pointers**.

```text
left →                ← right
```

This is used for sorted two-sum, valid palindrome, reversing arrays, container with most water, and 3Sum after sorting.

The second is **same-direction pointers**.

```text
slow →
fast →
```

This is used for removing duplicates, moving zeros, in-place filtering, and array compression.

The third is **fast and slow pointers**.

```text
slow moves one step
fast moves two steps
```

This is common in linked list cycle detection and finding the middle of a linked list.

The fourth is **sliding window**.

```text
left → window → right
```

This is used for substring and subarray optimization problems.

The key idea is:

**Identify what each pointer represents before writing code.**

**(PPT: Four Two Pointer Patterns)**

## PPT 6 — Opposite Direction Example: Two Sum Sorted

Let’s start with opposite-direction pointers.

For a sorted two-sum problem, we place one pointer at the beginning and one pointer at the end.

```python
def two_sum_sorted(nums, target):
    left = 0
    right = len(nums) - 1

    while left < right:
        total = nums[left] + nums[right]

        if total == target:
            return [left, right]
        elif total < target:
            left += 1
        else:
            right -= 1

    return [-1, -1]
```

The reasoning is simple.

If the sum is too small, we need a larger number, so we move the left pointer right.

If the sum is too large, we need a smaller number, so we move the right pointer left.

Since the array is sorted, this movement is safe.

The key interview explanation is:

**Because moving pointers changes the sum in a predictable direction, we can eliminate impossible pairs without checking all combinations.**

**(PPT: Two Sum Sorted Code + Movement Rule)**

## PPT 7 — Same Direction Example: Remove Duplicates

Now let’s look at same-direction pointers.

This pattern is often used for in-place modification.

Usually, the fast pointer scans the array, and the slow pointer tracks the next position to write a valid value.

For example, removing duplicates from a sorted array:

```python
def remove_duplicates(nums):
    if not nums:
        return 0

    slow = 1

    for fast in range(1, len(nums)):
        if nums[fast] != nums[fast - 1]:
            nums[slow] = nums[fast]
            slow += 1

    return slow
```

Here, fast scans every element.

Slow marks where the next unique value should be placed.

Because the array is sorted, duplicates are adjacent.

So whenever the current value differs from the previous value, we keep it.

The key idea is:

**Same-direction pointers are useful when one pointer reads and the other pointer writes.**

**(PPT: Fast Reads, Slow Writes)**

## PPT 8 — Fast and Slow Pointers

Fast and slow pointers are most common in linked list problems.

The classic example is cycle detection.

```python
def has_cycle(head):
    slow = head
    fast = head

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

        if slow == fast:
            return True

    return False
```

The slow pointer moves one step at a time.

The fast pointer moves two steps at a time.

If the linked list has a cycle, the fast pointer will eventually catch the slow pointer.

If there is no cycle, the fast pointer will reach the end.

This same idea can also be used to find the middle of a linked list.

The key idea is:

**Fast and slow pointers reveal structural properties of linked data.**

**(PPT: Fast Slow Pointer — Cycle Detection)**

## PPT 9 — Sliding Window as Two Pointers

Sliding window is also a two-pointer pattern.

The right pointer expands the window.

The left pointer shrinks the window when the current window becomes invalid.

A common example is finding the longest substring without repeating characters.

```python
def length_of_longest_substring(s):
    seen = set()
    left = 0
    best = 0

    for right in range(len(s)):
        while s[right] in seen:
            seen.remove(s[left])
            left += 1

        seen.add(s[right])
        best = max(best, right - left + 1)

    return best
```

The invariant is:

```text
The current window has no duplicate characters.
```

When the right pointer adds a duplicate character, we move the left pointer until the window becomes valid again.

The key idea is:

**Sliding window works when we can update the window state incrementally.**

**(PPT: Sliding Window — Expand and Shrink)**

## PPT 10 — Senior-Level Explanation and Closing Insight

At senior level, the most important part is not memorizing templates.

The most important part is explaining correctness.

For any two-pointer problem, I ask five questions.

What does each pointer represent?

What invariant do we maintain?

What condition controls pointer movement?

Why does moving this pointer not skip a valid answer?

And what are the time and space trade-offs?

Most two-pointer scans are **O(n)** because each pointer moves monotonically and never moves backward.

Space is usually **O(1)**, especially for in-place array problems.

But there are trade-offs.

Some problems require sorted input.

Sorting may break original index requirements.

Duplicates need careful handling.

And pointer boundaries can easily cause bugs.

To summarize:

Two pointers is an algorithmic technique for scanning data efficiently with two moving references.

It is useful for arrays, strings, linked lists, pair search, in-place modification, cycle detection, and window-based optimization.

The final insight is:

**Two Pointers is not about using two indexes.
It is about finding a safe movement rule that reduces the search space without losing valid answers.**

Thank you.
