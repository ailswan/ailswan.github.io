Hello everyone.

BFS stands for **Breadth-First Search**.

It is a traversal algorithm that explores a structure level by level.

The basic idea is:

```text
Start from a node
→ Visit all nodes one step away
→ Then all nodes two steps away
→ Then all nodes three steps away
```

That is why BFS is commonly used for shortest path in unweighted graphs, minimum steps, level-order traversal, grid problems, and multi-source spreading problems.

The key idea is:

**BFS explores by distance.**

## PPT 3 — Why BFS Works

BFS uses a queue.

A queue follows **First In, First Out** behavior.

```text
push A
push B
push C

pop → A
pop → B
pop → C
```

This order matters because nodes discovered earlier are processed earlier.

If we start from a node `S`, BFS explores like this:

```text
Level 0: S
Level 1: neighbors of S
Level 2: neighbors of level 1
Level 3: neighbors of level 2
```

So in an unweighted graph, where every edge has the same cost, the first time BFS reaches a node, it has found the shortest path to that node.

The key idea is:

**BFS works because it processes all nodes at distance d before distance d plus one.**

**(PPT: Queue + Level-by-Level Expansion)**

## PPT 4 — BFS vs DFS

BFS and DFS are both traversal algorithms, but they explore in different orders.

BFS uses a queue and explores level by level.

DFS uses recursion or a stack and explores deeply first.

BFS is usually better for:

```text
shortest path in unweighted graph
minimum number of steps
level-order traversal
nearest target
```

DFS is usually better for:

```text
backtracking
exploring all possibilities
connected components
cycle detection
path existence
```

So if the problem says “minimum steps,” “shortest path,” “nearest,” or “level order,” BFS is usually a strong candidate.

The key idea is:

**BFS is for minimum distance. DFS is for deep exploration.**

## PPT 5 — Basic BFS Template

A standard BFS needs five things.

A queue.
A visited set.
A start node.
A way to generate neighbors.
And processing logic.

Here is the basic graph BFS template:

```python
from collections import deque

def bfs(start, graph):
    queue = deque([start])
    visited = set([start])

    while queue:
        node = queue.popleft()

        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
```

The visited set is important because graphs can have cycles.

Without visited, we may keep visiting the same nodes forever.

A key detail is that I usually mark visited when enqueueing, not when dequeueing.

This prevents the same node from being added to the queue multiple times.

**(PPT: Basic BFS Template)**

## PPT 6 — Level-Order BFS on Trees

BFS is commonly used for binary tree level-order traversal.

For example:

```text
    3
   / \
  9  20
     / \
    15  7
```

The result is:

```text
[[3], [9, 20], [15, 7]]
```

The code is:

```python
from collections import deque

def level_order(root):
    if not root:
        return []

    result = []
    queue = deque([root])

    while queue:
        size = len(queue)
        level = []

        for _ in range(size):
            node = queue.popleft()
            level.append(node.val)

            if node.left:
                queue.append(node.left)

            if node.right:
                queue.append(node.right)

        result.append(level)

    return result
```

The key idea is:

**At the beginning of each loop, the queue contains exactly one level.**

**(PPT: Binary Tree Level Order)**

## PPT 7 — Shortest Path in Unweighted Graph

BFS is especially useful for shortest path in an unweighted graph.

We store both the node and its distance.

```python
from collections import deque

def shortest_path(graph, start, target):
    queue = deque([(start, 0)])
    visited = set([start])

    while queue:
        node, distance = queue.popleft()

        if node == target:
            return distance

        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, distance + 1))

    return -1
```

Because BFS processes nodes in increasing distance order, the first time we reach the target must be the shortest path.

This only works when every edge has the same cost.

If edge weights are different, plain BFS is not enough.

The key idea is:

**Plain BFS gives shortest path only in unweighted graphs.**

**(PPT: Unweighted Shortest Path)**

## PPT 8 — Grid BFS

Many LeetCode BFS problems happen on a 2D grid.

In grid BFS, each cell is a graph node.

Neighbors are generated using direction arrays.

```python
directions = [
    (1, 0),
    (-1, 0),
    (0, 1),
    (0, -1)
]
```

For each neighbor, we check:

```text
Is it inside the grid?
Is it not blocked?
Is it not visited?
```

This pattern appears in Number of Islands, Shortest Path in Binary Matrix, Rotting Oranges, Walls and Gates, and nearest distance problems.

For Number of Islands, every time we find an unvisited land cell, we start BFS and mark the whole island as visited.

The key idea is:

**Grid BFS is just graph BFS where each cell is a node.**

**(PPT: Grid BFS — Cells as Nodes)**

## PPT 9 — Multi-Source BFS and BFS With State

Sometimes BFS starts from many sources at once.

This is called **multi-source BFS**.

For example, in Rotting Oranges, all rotten oranges spread at the same time.

So we put all initially rotten oranges into the queue before BFS starts.

Each BFS level represents one minute.

Another example is 01 Matrix.

Instead of starting from every 1 and searching for the nearest 0, we start from all 0 cells at once.

BFS expands outward, so the first time a cell is reached, it has the shortest distance to a 0.

Some BFS problems also require state beyond position.

For example:

```text
(row, col, remaining_obstacle_eliminations)
(node, keys_collected)
(position, used_special_move)
```

In those cases, visited must include the full state, not just the position.

The key idea is:

**BFS state must include everything that affects future choices.**

## PPT 10 — Senior-Level Explanation and Closing Insight

At senior level, I would explain BFS like this.

BFS is a traversal algorithm that explores nodes level by level using a queue.

The queue represents the current frontier.

A visited set prevents cycles and duplicate work.

For unweighted graphs, BFS finds shortest path because it explores all nodes at distance `d` before exploring distance `d + 1`.

For tree problems, BFS is natural for level-order traversal.

For grid problems, each cell can be modeled as a node, and directions generate neighbors.

For multi-source problems, we put all sources into the queue initially, so expansion happens simultaneously.

For stateful BFS, the visited key must include the full state, not just the current node.

The complexity for graph BFS is usually **O(V + E)**.

For grid BFS, it is usually **O(mn)**.

The final insight is:

**BFS is not just using a queue.
It is level-by-level expansion.**

If you can explain what the node is, how neighbors are generated, what visited contains, what each level means, and why first arrival gives the shortest path, then you understand BFS at an interview-ready level.

Thank you.
