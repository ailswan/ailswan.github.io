## PPT 1 — Title Slide

Hello everyone.
Today I’d like to walk through another classic LeetCode pattern: **Fast and Slow Pointer**.

Fast and slow pointer is a two-pointer technique where two pointers move through a sequence at different speeds.

Usually:

```text
slow pointer moves one step
fast pointer moves two steps
```

The basic idea is:

If there is a cycle, the fast pointer will eventually catch the slow pointer.

If there is no cycle, the fast pointer will eventually reach the end.

The key insight is:

**Fast and slow pointer uses relative speed to detect hidden structure in a sequence.**

## PPT 3 — Why Fast and Slow Pointer Works

The intuition is like two runners on a circular track.

One runner is slow and moves one step at a time.

The other runner is fast and moves two steps at a time.

If the track is circular, the fast runner will eventually catch the slow runner.

If the path is not circular, the fast runner simply reaches the end.

In linked list terms:

```text
Acyclic list:
1 → 2 → 3 → 4 → null
```

Fast eventually reaches null.

But in a cyclic list, fast keeps moving inside the cycle and eventually meets slow.

The key idea is:

**Inside a cycle, fast gains one node on slow at each step, so they must eventually meet.**

**(PPT: Runner Analogy + Linked List Cycle)**

## PPT 4 — Floyd’s Cycle Detection

The classic algorithm is called **Floyd’s Cycle Detection**, also known as the **Tortoise and Hare Algorithm**.

The slow pointer is the tortoise.

The fast pointer is the hare.

The code looks like this:

```python
def has_cycle(head):
    slow = head
    fast = head

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

        if slow == fast:
            return True

    return False
```

The condition:

```text
while fast and fast.next
```

is important because fast moves two steps.

If fast reaches null, there is no cycle.

If slow and fast meet, there is a cycle.

The key idea is:

**Meeting means cycle. Null means no cycle.**

**(PPT: Floyd Cycle Detection Code)**

## PPT 5 — Finding the Middle of a Linked List

Fast and slow pointer is also useful for finding the middle of a linked list.

For example:

```text
1 → 2 → 3 → 4 → 5
middle = 3
```

And for even length:

```text
1 → 2 → 3 → 4 → 5 → 6
middle = 4
```

The code is:

```python
def middle_node(head):
    slow = head
    fast = head

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

    return slow
```

Why does this work?

Because fast moves twice as fast as slow.

So when fast reaches the end, slow has traveled half the distance.

One interview detail is important:

For even-length lists, this common template returns the **second middle**.

The key idea is:

**Fast reaches the end when slow reaches the middle.**

**(PPT: Middle of Linked List)**

## PPT 6 — Cycle Entry Point

Now let’s look at a harder problem: finding where the cycle begins.

The algorithm has two phases.

First, use fast and slow pointers to detect a meeting point inside the cycle.

Second, move one pointer back to the head, and keep the other pointer at the meeting point.

Then move both one step at a time.

They will meet at the cycle entry.

```python
def detect_cycle(head):
    slow = head
    fast = head

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

        if slow == fast:
            ptr1 = head
            ptr2 = slow

            while ptr1 != ptr2:
                ptr1 = ptr1.next
                ptr2 = ptr2.next

            return ptr1

    return None
```

The key point is:

**The first meeting point is not necessarily the cycle entry.**

The second phase is required to find the actual entry node.

**(PPT: Cycle Entry — Two Phases)**

## PPT 7 — Why Cycle Entry Logic Works

The proof is based on distances.

Let:

```text
a = distance from head to cycle entry
b = distance from cycle entry to meeting point
c = distance from meeting point back to cycle entry
```

The cycle length is:

```text
b + c
```

When slow and fast meet, fast has traveled twice as far as slow.

The distance difference between them is a multiple of the cycle length.

This leads to the key relationship:

```text
distance from head to entry
equals
distance from meeting point to entry
mod cycle length
```

So if one pointer starts from head, and another starts from the meeting point, and both move at the same speed, they meet at the cycle entry.

The key idea is:

**Relative speed finds the cycle. Equal speed finds the cycle entry.**

**(PPT: Cycle Entry Distance Proof)**

## PPT 8 — Beyond Linked Lists: Happy Number

Fast and slow pointer is not only for linked lists.

It works whenever the problem can be modeled as a sequence of states.

A good example is **Happy Number**.

A happy number repeatedly replaces the number with the sum of squares of its digits.

For example:

```text
19
→ 82
→ 68
→ 100
→ 1
```

So 19 is happy.

But if the sequence does not reach 1, it eventually repeats, which means it forms a cycle.

```python
def is_happy(n):
    def next_number(x):
        total = 0

        while x > 0:
            digit = x % 10
            total += digit * digit
            x //= 10

        return total

    slow = n
    fast = next_number(n)

    while fast != 1 and slow != fast:
        slow = next_number(slow)
        fast = next_number(next_number(fast))

    return fast == 1
```

The key idea is:

**A repeated state means a cycle, even when the structure is not a linked list.**

**(PPT: Happy Number as Cycle Detection)**

## PPT 9 — Find Duplicate Number

Another advanced example is **Find Duplicate Number**.

The constraint is:

```text
array length = n + 1
values are from 1 to n
```

This means we can treat the array as an implicit linked list.

Each index points to another index:

```text
index → nums[index]
```

Because there is a duplicate, two paths point into the same value, which creates a cycle.

```python
def find_duplicate(nums):
    slow = nums[0]
    fast = nums[0]

    while True:
        slow = nums[slow]
        fast = nums[nums[fast]]

        if slow == fast:
            break

    ptr1 = nums[0]
    ptr2 = slow

    while ptr1 != ptr2:
        ptr1 = nums[ptr1]
        ptr2 = nums[ptr2]

    return ptr1
```

The cycle entry corresponds to the duplicate number.

The key idea is:

**A duplicate value creates a cycle in the implicit index-to-value graph.**

**(PPT: Find Duplicate Number as Implicit Linked List)**

## PPT 10 — Senior-Level Explanation and Closing Insight

At senior level, I would explain Fast and Slow Pointer like this.

Fast and slow pointer is a two-pointer technique where two pointers move through a sequence at different speeds.

It works especially well when the structure is deterministic.

That means each state points to exactly one next state.

If the sequence has no cycle, fast eventually reaches the end.

If the sequence has a cycle, fast eventually catches slow because it gains one step each iteration.

This gives us **O(n)** time and **O(1)** extra space.

It is useful for linked list cycle detection, finding the middle of a linked list, finding cycle entry, Happy Number, and Find Duplicate Number.

But it does not fit every problem.

If the structure branches, or each state can have multiple next states, then the problem is more like graph traversal, and BFS or DFS may be more appropriate.

The final insight is:

**Fast and Slow Pointer is not just about one pointer being faster.
It is about using relative speed to detect cycles or structural positions in a deterministic sequence.**

If you can explain why fast catches slow, why the first meeting point is not always the entry, and why the method uses constant space, then you understand this pattern at an interview-ready level.

Thank you.

**(PPT: Closing Insight — Relative Speed + Cycle Detection)**
