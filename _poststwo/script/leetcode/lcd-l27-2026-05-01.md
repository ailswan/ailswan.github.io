## PPT 3 — Top-Down Recursion

Top-down recursion passes information from parent to child.

Examples of state include:

```text
current path sum
current depth
current min and max bounds
current path list
```

The template looks like this:

```python
def dfs(node, state_from_parent):
    if not node:
        return

    new_state = update(state_from_parent, node)

    dfs(node.left, new_state)
    dfs(node.right, new_state)
```

This is useful when the current node needs information from its ancestors.

Common examples include Path Sum, Path Sum II, root-to-leaf paths, and Validate BST with bounds.

For example, in Path Sum, we pass the remaining target downward.

At each node, we subtract the current value.

The key idea is:

**Top-down recursion carries context from ancestors to descendants.**

**(PPT: Top-Down Recursion — Pass State Downward)**

## PPT 4 — Bottom-Up Recursion

Bottom-up recursion is the opposite.

Here, the parent needs information from its children first.

Examples include:

```text
height
diameter
balanced status
max path contribution
subtree size
subtree sum
```

The template is:

```python
def dfs(node):
    if not node:
        return base_value

    left = dfs(node.left)
    right = dfs(node.right)

    return combine(left, right, node)
```

This pattern appears in Maximum Depth, Balanced Binary Tree, Diameter of Binary Tree, Maximum Path Sum, Count Nodes, and Lowest Common Ancestor.

For maximum depth, each node returns:

```text
1 + max(left depth, right depth)
```

The key idea is:

**Bottom-up recursion returns information from children back to the parent.**

**(PPT: Bottom-Up Recursion — Return Subtree Info)**

## PPT 5 — Height, Balance, and Diameter

Let’s look at three classic bottom-up examples.

For maximum depth:

```python
def max_depth(root):
    if not root:
        return 0

    left = max_depth(root.left)
    right = max_depth(root.right)

    return 1 + max(left, right)
```

For Balanced Binary Tree, we return height if the subtree is balanced, and `-1` if it is not balanced.

That lets us stop early when a subtree already fails.

For Diameter of Binary Tree, at each node, the longest path passing through that node is:

```text
left_height + right_height
```

We return height upward, but update a global answer for the diameter.

The key idea is:

**Many binary tree problems return one value upward while updating another answer globally.**

**(PPT: Depth + Balance + Diameter)**

## PPT 6 — Maximum Path Sum and Global Answer

Maximum Path Sum is a very important global-answer pattern.

A path can start and end at any nodes, but it must be connected.

At each node, there are two concepts.

First, the best path using the current node as the highest point:

```text
node.val + left_gain + right_gain
```

Second, the value returned to the parent:

```text
node.val + max(left_gain, right_gain)
```

The returned value can only choose one side.

Why?

Because a path returned to the parent cannot split into both left and right branches.

```python
def max_path_sum(root):
    answer = float("-inf")

    def gain(node):
        nonlocal answer

        if not node:
            return 0

        left_gain = max(gain(node.left), 0)
        right_gain = max(gain(node.right), 0)

        answer = max(answer, node.val + left_gain + right_gain)

        return node.val + max(left_gain, right_gain)

    gain(root)
    return answer
```

The key idea is:

**The answer candidate may use both sides, but the return value to parent can only use one side.**

**(PPT: Maximum Path Sum — Candidate vs Return Value)**

## PPT 7 — Path Problems and Backtracking

Path problems are usually top-down DFS.

For Path Sum, we pass the remaining target downward.

```python
def has_path_sum(root, target_sum):
    if not root:
        return False

    if not root.left and not root.right:
        return target_sum == root.val

    remaining = target_sum - root.val

    return (
        has_path_sum(root.left, remaining)
        or
        has_path_sum(root.right, remaining)
    )
```

For Path Sum II or Binary Tree Paths, we need to maintain a current path.

That becomes backtracking.

We append the current node, explore children, and then pop before returning.

The important detail is:

```python
result.append(path[:])
```

We append a copy of the path, not the same mutable list.

The key idea is:

**Path problems pass state downward and often require backtracking.**

**(PPT: Path Sum + Backtracking)**

## PPT 8 — Tree Comparison and LCA

Some binary tree problems compare structure.

For Same Tree, we compare two nodes recursively.

If both are null, they match.

If only one is null, they do not match.

If values differ, they do not match.

Then we compare left with left and right with right.

For Symmetric Tree, we compare mirror positions:

```text
left.left with right.right
left.right with right.left
```

For Lowest Common Ancestor in a general binary tree, we use postorder recursion.

If the current node is `p` or `q`, return it.

If left and right both return non-null, the current node is the LCA.

Otherwise, return the non-null side.

The key idea is:

**Tree comparison problems are recursion over structure, not just values.**

**(PPT: Same Tree + Symmetric Tree + LCA)**

## PPT 9 — BST and Level Order Patterns

BST problems have special structure.

For a valid BST:

```text
left subtree values < node.val
right subtree values > node.val
```

One way to validate a BST is bounds recursion.

Each node must be within the valid range passed down from its ancestors.

Another way is inorder traversal.

For a valid BST, inorder traversal gives a strictly increasing sequence.

This is also why Kth Smallest in BST uses inorder traversal.

Level-order problems use BFS.

Examples include Level Order Traversal, Right Side View, Zigzag Level Order, and sometimes Minimum Depth.

The key idea is:

**BST problems use ordering rules; level problems use BFS.**

**(PPT: BST Bounds / Inorder + BFS Level Order)**

## PPT 10 — Senior-Level Explanation and Closing Insight

At senior level, I would explain binary tree patterns like this.

Binary tree problems are usually about recursively processing a node, its left subtree, and its right subtree.

The first step is identifying the pattern.

If information flows from parent to child, it is top-down recursion.

If the parent needs child results first, it is bottom-up recursion.

If the problem asks for paths, I usually use DFS with backtracking.

If the problem is BST-specific, I use inorder traversal or bounds recursion.

If the problem asks for levels, right side view, or zigzag order, I use BFS.

Some problems need a global answer.

For example, Diameter returns height upward while updating the longest path globally.

Maximum Path Sum returns one-side gain upward while updating the best path globally.

The final insight is:

**Binary Tree Patterns are not just “see a tree, write recursion.”
They are about identifying what each node receives, what it returns, and whether the answer is local, global, path-based, BST-based, or level-based.**

If you can explain the null base case, the recursive return value, top-down versus bottom-up flow, path backtracking, BST rules, BFS levels, and edge cases like empty trees, skewed trees, negative values, and duplicates, then you understand binary tree patterns at an interview-ready level.
